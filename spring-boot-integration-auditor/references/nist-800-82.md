# NIST SP 800-82 Rev 3 — ICS/OT Security Mapping to Spring Boot

> Part of the `spring-boot-integration-auditor` skill — back to [SKILL.md](../SKILL.md)

NIST SP 800-82 "Guide to Operational Technology (OT) Security" Rev 3 (September 2023) provides guidance for securing Industrial Control Systems (ICS). This file maps the most relevant SP 800-82 controls and SP 800-53 underlying controls to Spring Boot integration patterns when the application sits at the IT/OT boundary.

**When this reference applies**: Spring Boot integration that touches OT — OPC UA, Modbus, MQTT broker industrial, SCADA gateways, historians, PLCs. For pure IT/enterprise Spring Boot apps, the relevance of 800-82 is limited to ICS-adjacent checks (network segmentation, integrity, monitoring).

## AC — Access Control

### AC-2, AC-3 — Account Management / Access Enforcement

**Spring Boot check**: every OT-facing endpoint enforces authentication and authorization.

```bash
# Endpoints without security
rg -n "@(GetMapping|PostMapping|PutMapping|DeleteMapping|PatchMapping)" src/ --type java \
  | rg -v "@PreAuthorize|@Secured|@RolesAllowed"

# Missing role checks
rg -in "(plant|scada|plc|historian|ot).*endpoint" src/ --type java \
  | rg -v "hasRole|hasAuthority"
```

| # | Vulnerable | Secure |
|---|-----------|--------|
| 1 | `@GetMapping("/api/plc/tags")` no `@PreAuthorize` | `@PreAuthorize("hasRole('OT_OPERATOR')")` |
| 2 | Default Spring Security (auto-generated user `user`) | Explicit user store + BCrypt + lockout |
| 3 | Shared service account for all OT clients | Per-client certificate + unique service account |

```java
// Secure OT endpoint
@PreAuthorize("hasRole('OT_OPERATOR') and @plantAccess.canRead(authentication, #plantId)")
@GetMapping("/api/plc/{plantId}/tags")
public List<Tag> readTags(@PathVariable String plantId) { ... }
```

### AC-4 — Information Flow Enforcement (Network Segmentation)

**Spring Boot check**: the application enforces segmentation between IT and OT zones.

| Check | How |
|-------|-----|
| IT/OT zone separation enforced at app level | Separate Spring profiles (`it`, `ot`) with different SecurityFilterChain |
| Outbound traffic to OT limited by allowlist | `application-ot.yml` with explicit `rest.template.connect-timeout` + custom interceptor for host allowlist |
| No implicit route to internet from OT zone | `management.metrics.export.prometheus.enabled=false` in OT profile, no cloud SDKs |
| mTLS for OT-zone traffic | `server.ssl.client-auth=need` + truststore with OT CA only |

```yaml
# application-ot.yml — strict, no internet
server:
  port: 8443
  ssl:
    enabled: true
    client-auth: need  # mTLS required
    trust-store: classpath:ot-truststore.p12
    trust-store-password: ${OT_TRUSTSTORE_PWD}
  # NO outbound to cloud / internet
```

### AC-5 — Separation of Duties

**Spring Boot check**: write operations to OT require separate role from read.

```java
@PreAuthorize("hasRole('OT_ENGINEER')")        // read
@GetMapping("/api/plc/{id}/program")
public Program read(@PathVariable String id) { ... }

@PreAuthorize("hasRole('OT_ENGINEER_LEAD') and @approval.isApproved(#changeId)")  // write
@PutMapping("/api/plc/{id}/program")
public Program write(@PathVariable String id, @RequestBody Program p) { ... }
```

### AC-6 — Least Privilege

**Spring Boot check**: default-deny posture, deny-all base, explicit allow.

```java
http.authorizeHttpRequests(auth -> auth
    .anyRequest().denyAll()  // default deny
    .requestMatchers("/api/ot/read/**").hasRole("OT_OPERATOR")
    .requestMatchers("/api/ot/write/**").hasRole("OT_ENGINEER")
    .requestMatchers("/actuator/health").permitAll()
);
```

### AC-17 — Remote Access

**Spring Boot check**: remote access to OT uses jump host / VPN, not direct internet exposure.

| Check | Evidence |
|-------|----------|
| App is NOT publicly accessible | Reverse proxy with IP allowlist, no public DNS |
| Admin via VPN + MFA | Gateway (WireGuard, OpenVPN, Cloudflare Tunnel) |
| Session recording for OT admin actions | `ApplicationEventPublisher` publishes to immutable audit log |

## AU — Audit and Accountability

### AU-2, AU-3 — Audit Events / Content of Audit Records

**Spring Boot check**: every OT-relevant state change produces an audit record with timestamp, actor, action, target.

```bash
# Audit event listeners
rg -n "AbstractAuthenticationAuditHandler|AuthenticationSuccessEvent|AuthenticationFailureEvent" src/ --type java

# Audit publishers
rg -n "AuditEvent|AuditEventPublisher|@EventListener" src/ --type java
```

| # | Missing audit | Severity |
|---|--------------|----------|
| 1 | Login / logout not logged | High |
| 2 | PLC write (setpoint change) not logged | Critical |
| 3 | Tag value override not logged | Critical |
| 4 | Alarm acknowledge not logged | High |
| 5 | User role change not logged | High |
| 6 | Configuration change not logged | High |

```java
// Secure audit event for OT action
@EventListener
public void onPlcWrite(PlcWriteEvent e) {
    AuditEvent ae = new AuditEvent(
        Instant.now(),
        e.getUsername(),
        "PLC_WRITE",
        Map.of(
            "plc", e.getPlcId(),
            "tag", e.getTag(),
            "oldValue", e.getOldValue(),
            "newValue", e.getNewValue(),
            "approval", e.getApprovalId()
        )
    );
    auditPublisher.publish(ae);  // immutable, signed, to SIEM
}
```

### AU-6 — Audit Review

**Spring Boot check**: logs flow to a SIEM with retention >= 90 days for OT environments.

| Check | How |
|-------|-----|
| Logback writes to JSON file | `LogstashEncoder` |
| File is append-only | `RollingFileAppender` with read-only mount or object lock (S3 Object Lock) |
| Forwarded to SIEM | Filebeat / Fluentd sidecar, or OTLP / Loki |
| Retention policy >= 90d | NERC-CIP requires 90 days minimum |

## CM — Configuration Management

### CM-6, CM-7 — Configuration Settings / Least Functionality

**Spring Boot check**: only required features are enabled; defaults are hardened.

```bash
# Actuator endpoints in prod
rg -n "management\.endpoints\.web\.exposure" src/main/resources/

# Dev / debug features
rg -in "spring\.devtools|@Profile\(.dev.\)|debug:\s*true" src/main/resources/

# Unused features
rg -in "spring\.mail|spring\.batch|quartz" src/main/resources/
```

| # | Misconfiguration | Severity |
|---|------------------|----------|
| 1 | `management.endpoints.web.exposure.include=*` | Critical (full actuator leak) |
| 2 | `spring.jpa.hibernate.ddl-auto=update` in prod | High (attacker can mutate schema) |
| 3 | `spring.devtools.livereload.enabled=true` in prod | High (RCE) |
| 4 | `server.servlet.session.timeout: -1` (never expire) | High |
| 5 | `management.endpoint.env.enabled=true` | High (secrets leak) |
| 6 | `management.endpoint.heapdump.enabled=true` (exposed) | High (memory dump = secrets) |
| 7 | Outbound HTTP enabled without allowlist | Medium |

### CM-8 — System Component Inventory

**Spring Boot check**: SBOM generated and updated per release.

```bash
# Generate SBOM
mvn org.cyclonedx:cyclonedx-maven-plugin:makeAggregateBom -DoutputFormat=json
mvn org.springframework.boot:spring-boot-starter-actuator:build-info

# Check SBOM exists
ls target/bom.json target/sbom.json 2>/dev/null
```

| # | Missing | Severity |
|---|---------|----------|
| 1 | No SBOM in build | High |
| 2 | SBOM not stored in artifact registry | High |
| 3 | No dependency tracking over time | Medium |
| 4 | Third-party OT libraries (Eclipse Milo, j2mod) not inventoried | High |

## IA — Identification and Authentication

### IA-2, IA-5 — User Identification / Authenticator Management

**Spring Boot check**: strong auth for all OT endpoints, no shared secrets in source.

| # | Vulnerable | Secure |
|---|-----------|--------|
| 1 | `withDefaultPasswordEncoder()` | BCrypt with cost >= 12, rotation |
| 2 | API key in `application.yml` | Vault, env, or Spring Cloud Config encrypted |
| 3 | Single password for all OT clients | Per-client certificate (mTLS) |
| 4 | `inMemoryAuthentication` in prod | LDAP/AD/OAuth2 with central IdP |

```java
// mTLS for OT clients
@Bean
SecurityFilterChain otFilterChain(HttpSecurity http) throws Exception {
    http.securityMatcher("/api/ot/**")
        .authorizeHttpRequests(a -> a.anyRequest().hasRole("OT_CLIENT"))
        .x509(x509 -> x509.subjectPrincipalRegex("CN=(.*?)(?:,|$)"))
        .sessionManagement(s -> s.sessionCreationPolicy(SessionCreationPolicy.STATELESS));
    return http.build();
}
```

### IA-8 — Identification and Authentication (Non-Organizational Users)

**Spring Boot check**: third-party integrators (vendor, contractor) use dedicated accounts with expiry.

| Check | Evidence |
|-------|----------|
| Temporary account with TTL | `accountExpiry` set in user entity |
| Vendor accounts segregated by namespace | LDAP OU for vendors |
| All access logged with vendor tag | `AuditEvent.actor` includes `vendor:<id>` |

## IR — Incident Response

### IR-4 — Incident Handling

**Spring Boot check**: app exposes hooks for IR (kill switch, force-logout, lock-down).

| # | Capability | Implementation |
|---|-----------|----------------|
| 1 | Force-logout all users | `SessionRegistry` + `expireSessions` |
| 2 | Disable write operations to OT | Feature flag `OT_WRITE_ENABLED=false` via env refresh |
| 3 | Trigger alarm to SIEM | `ApplicationEventPublisher` to Kafka / REST |
| 4 | Capture forensic snapshot (config, recent audit, current state) | Custom endpoint `/internal/forensic-snapshot` (admin only) |
| 5 | Read-only mode | `@ConditionalOnProperty("ot.read-only")` for OT write beans |

```java
@ConditionalOnProperty(name = "ot.write.enabled", havingValue = "true", matchIfMissing = true)
@Service
public class PlcWriteService { ... }

@ConditionalOnProperty(name = "ot.write.enabled", havingValue = "false")
@Service
public class PlcWriteServiceNoOp implements PlcWriteService { ... }
```

### IR-6 — Incident Reporting

**Spring Boot check**: alerts to SIEM with structured payload.

```yaml
# application-ot.yml
alerts:
  sink: kafka
  topic: ot-incidents
  severity-map:
    critical: P1
    high: P2
    medium: P3
```

## SC — System and Communications Protection

### SC-7 — Boundary Protection

**Spring Boot check**: app does not expose internal services externally.

| # | Check |
|---|-------|
| 1 | Actuator on separate port, not 8080 |
| 2 | No management endpoints in IT-facing network |
| 3 | Egress filtering: only approved hosts for outbound |
| 4 | Reverse proxy in front, hides Spring Boot |
| 5 | mTLS for OT-zone traffic |

### SC-8 — Transmission Confidentiality and Integrity

**Spring Boot check**: all OT traffic uses TLS 1.2+ (prefer 1.3) with strong ciphers.

```yaml
server:
  ssl:
    enabled: true
    protocol: TLS
    enabled-protocols: TLSv1.3
    ciphers: TLS_AES_256_GCM_SHA384,TLS_AES_128_GCM_SHA256
    key-store: classpath:keystore.p12
    key-store-password: ${KEYSTORE_PWD}
    key-store-type: PKCS12
    key-alias: app
```

| # | Vulnerable | Secure |
|---|-----------|--------|
| 1 | `server.port: 8080` no SSL | TLS 1.2+ enforced, port 8443 |
| 2 | `enabled-protocols: TLSv1,TLSv1.1` | TLSv1.2, TLSv1.3 only |
| 3 | Self-signed cert in prod | CA-signed, or internal CA in truststore |
| 4 | No `verifyClient` for OT | `need` (mTLS) |
| 5 | Plain OPC UA TCP (`opc.tcp://`) without security policy | `opc.tcp://host:4840` with `SecurityPolicy.Basic256Sha256` |

### SC-12 — Cryptographic Key Establishment and Management

**Spring Boot check**: keys in vault, not in repo.

| # | Check |
|---|-------|
| 1 | No private keys in `src/main/resources/` |
| 2 | Keystore password from env / vault |
| 3 | Key rotation policy (annual minimum for prod) |
| 4 | Separate keys per environment (dev, staging, prod) |

### SC-13 — Cryptographic Protection

**Spring Boot check**: FIPS 140-2/3 validated modules where required.

```bash
# If FIPS required, check JVM flag
java -Djava.security.properties=conf/fips.security
```

### SC-28 — Protection of Information at Rest

**Spring Boot check**: data at rest encrypted (DB, file uploads, audit logs).

| # | Check |
|---|-------|
| 1 | DB at-rest encryption (TDE / cloud-native) |
| 2 | File uploads encrypted |
| 3 | Audit log file encrypted or write-once storage |
| 4 | Secrets encrypted (Jasypt, Spring Cloud Config server, Vault) |

| High CVE | 30 days |

### SI-4 — Information System Monitoring

**Spring Boot check**: app emits security-relevant metrics.

```bash

rg -n "MeterRegistry|Counter\.builder|Timer\.builder" src/ --type java | head -20

# Spring Boot Actuator metrics
```

- `ot.plc.write.count` (counter)
- `ot.plc.write.errors` (counter)
- `ot.authentication.failures` (counter)

## ICS-Specific Protocol Checks (Eclipse Milo, HiveMQ, j2mod)

When the Spring Boot integration uses these libraries, additional protocol-level checks apply:

### OPC UA (Eclipse Milo)

```bash
# Security policy
rg -in "SecurityPolicy\.(None|Basic128|Basic256|Basic256Sha256)" src/ --type java
```

| # | Vulnerable | Secure |
|---|-----------|--------|
| 1 | `SecurityPolicy.None` | `Basic256Sha256` or `Aes256Sha256RsaPss` (newer) |
| 2 | `MessageSecurityMode.None` | `SignAndEncrypt` |
| 3 | Anonymous authentication | Certificate-based client auth |
| 4 | No certificate validation | `ApplicationDescription.certificateRequired = true` |
| 5 | `EndpointConfiguration.discoveryOnly` for write endpoints | `discoveryOnly = false` only for legit clients |

### MQTT (HiveMQ / Paho)

```bash
rg -in "MqttConnectOptions|setSocketFactory|SSLContext" src/ --type java
```

| # | Vulnerable | Secure |
|---|-----------|--------|
| 1 | Plain `tcp://` (port 1883) | `ssl://` (port 8883) with mTLS |
| 2 | `setUsername`/`setPassword` (no cert) | Client certificate only |
| 3 | QoS 0 for OT commands | QoS 1 or 2 for OT commands |
| 4 | No `cleanSession=false` | `cleanSession=false` for OT clients |
| 5 | No topic-level ACLs | `acl` file per client cert, deny by default |

### Modbus (j2mod)

| # | Vulnerable | Secure |
|---|-----------|--------|
| 1 | Plain TCP Modbus (port 502) | Modbus/TLS (port 802) where possible |
| 2 | No unit ID allowlist | Validate unit ID per request |
| 3 | No function code allowlist | Reject function codes not used by app |
| 4 | Direct exposure to internet | Modbus gateway / DMZ only |

## Compliance Scorecard (NIST 800-82)

For the report, compute:

| Status | Meaning |
|--------|---------|
| Compliant | Control fully implemented and verified |
| Partial | Control implemented but with gaps |
| Non-compliant | No evidence of control |
| N/A | Control not applicable to this integration |

For each control, cite:
- File:line of the implementation
- Or `MISSING` if not found
- Or `N/A` with reason
